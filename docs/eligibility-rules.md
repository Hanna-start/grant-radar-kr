# 판정 규칙 문서

아직 규칙이 구현되지 않았다. 규칙은 실제 API 응답 구조를 확인한 뒤
(5단계에서) 구현하며, 각 규칙은 이 문서에 다음 형식으로 기록한다.

## 기록 형식

- 규칙 ID (예: `region.v1`)
- 목적
- 입력값 (공고 필드, 회사 필드)
- 판정 조건 (`PASS` / `FAIL` / `REVIEW` / `NOT_APPLICABLE` / `ERROR`)
- 자동 제외(`INELIGIBLE`) 허용 여부
- 예외 상황
- 테스트 사례
- 근거가 되는 정책 또는 공고 문구

## 공통 원칙

- 정보가 없거나 표현이 모호하면 `FAIL`이 아니라 `REVIEW`로 처리한다.
- 자동 제외를 허용하는 규칙은 별도 목록으로 관리한다. 모든 규칙의 `FAIL`이
  자동으로 전체 제외를 의미하지 않는다.
- 문자열 포함 여부만으로 판정하지 않는다 (지역·업종은 매핑표 사용).
- 경계값(`미만` / `이내` / `이하`)을 같은 의미로 취급하지 않는다.

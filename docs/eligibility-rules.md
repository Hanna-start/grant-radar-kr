# 판정 규칙 문서

각 규칙의 판정 조건과 자동 제외 허용 여부를 기록한다. 구현: `src/grant_radar/rules/`.

## 공통 원칙

- 정보가 없거나 표현이 모호하면 `FAIL`이 아니라 `REVIEW`로 처리한다.
- 자동 제외(`INELIGIBLE`)는 자동 제외 허용 목록(`AUTO_EXCLUDE_RULE_IDS`,
  `services/evaluation.py`)에 있는 규칙의 신뢰도 HIGH `FAIL`만 유발할 수 있다.
  목록 밖 규칙의 `FAIL`은 `REVIEW_REQUIRED`로 낮춘다.
- 문자열 포함 여부만으로 판정하지 않는다 (지역은 명시적 매핑표 사용).
- 경계값 표현(`미만`/`이내`/`이하`)을 같은 의미로 취급하지 않는다.
- 모든 결과에 공고 값, 회사 값, 이유, 근거 필드(evidence_field)를 포함한다.

## region.v1 — 지역 규칙

- **목적**: 공고 대상 지역과 회사 소재지의 명확한 일치/불일치 판정
- **입력값**: 공고 `supt_regin` (실제 관찰: "전국", "서울", "경기" 등 짧은 시도명),
  회사 `headquarters_region`, `business_locations`
- **판정 조건**:
  - 전국 또는 본점 소재지가 대상 지역에 포함 → `PASS`
  - 본점 불일치·사업장 일치 → `REVIEW` (본점/사업장 요건 확인 필요)
  - 매핑표(`data/reference/region_mapping.json`)로 해석 불가한 표현,
    지역 정보 없음, 회사 지역 정보 없음 → `REVIEW`
  - 본점·사업장 모두 불포함 → `FAIL` (HIGH)
- **자동 제외 허용**: 예 (FAIL HIGH 시)
- **예외**: "이전 예정 기업 허용" 등 조건부 허용은 supt_regin 필드에 없으므로,
  FAIL 시 human_checks로 본문 확인을 요구한다
- **테스트**: `tests/test_region_rule.py`
- **근거**: 지시서 14.1절, 실제 응답 관찰(2026-07-21)

## business_age.v1 — 업력 규칙

- **목적**: 공고의 업력(창업 기간) 허용 구간과 회사 업력의 비교
- **입력값**: 공고 `biz_enyy` (실제 관찰: "예비창업자,1년미만,...,10년미만" 쉼표
  구분 목록), 접수 시작/종료일, 회사 `established_date`
- **판정 조건**:
  - 접수 종료일 기준 업력이 나열된 최대 상한보다 작음 → `PASS`
    (접수 종료일 이전의 어떤 기준일에서도 성립. 정확한 날짜 연산 사용,
    "N년미만"은 설립일+N년이 되는 날부터 불성립)
  - 허용 대상이 예비창업자뿐인데 회사가 이미 설립됨 → `FAIL` (HIGH, 기준일 무관)
  - 경계 근처(접수 기간 중 상한 도달), 상한 초과로 보이지만 기준일 미상,
    "이내"·"이하" 등 해석 불가 표현, 설립일·종료일 정보 부족 → `REVIEW`
  - 조건 없음 → `NOT_APPLICABLE` (+본문 확인 항목)
- **자동 제외 허용**: 예 — 단, 기준일이 데이터에 없으므로 실질적으로
  예비창업자 전용 충돌만 FAIL이 된다. 업력 초과는 REVIEW로만 처리한다
  (지시서 14.2: 기준일이 없으면 임의로 결정하지 않는다)
- **예외**: 재창업·법인 전환 등 업력 산정 예외는 human_checks로 안내
- **테스트**: `tests/test_business_age_rule.py`
- **근거**: 지시서 14.2절, 실제 응답 관찰(2026-07-21)

## applicant_type.v1 — 신청자 유형 규칙

- **목적**: 공고 신청 대상 범주와 회사 형태의 명확한 충돌 판정
- **입력값**: 공고 `aply_trgt` (실제 관찰: "청소년,대학생,일반인,대학,연구기관,
  일반기업,1인 창조기업" 쉼표 구분 목록), 회사 `business_type`, `employee_count`
- **판정 조건**:
  - 회사가 명확히 해당하는 범주(법인→"일반기업", 직원 1인→"1인 창조기업" 등)가
    목록에 있음 → `PASS`
  - 목록 전체가 명백히 기업이 아닌 범주(청소년/대학생/대학/연구기관/공공기관)
    → `FAIL` (HIGH)
  - "일반인" 등 회사 해당 여부가 불명확한 범주만 남음, 회사 형태 정보 없음
    → `REVIEW`
  - 목록 없음 → `NOT_APPLICABLE` (+본문 확인 항목)
- **자동 제외 허용**: 예 (FAIL HIGH 시)
- **예외**: 대표자 개인 자격 신청 가능성은 판정하지 않으며 본문 확인에 맡긴다
- **테스트**: `tests/test_applicant_type_rule.py`
- **근거**: 지시서 14.3절, 실제 응답 관찰(2026-07-21)

## 전체 판정 (services/evaluation.py)

1. 자동 제외 허용 규칙의 신뢰도 HIGH `FAIL` ≥ 1 → `INELIGIBLE`
2. 그 외 `FAIL`·`REVIEW`·`ERROR` 존재 → `REVIEW_REQUIRED`
3. `PASS` ≥ 1이고 나머지 전부 `NOT_APPLICABLE` → `ELIGIBLE`
4. 적용 가능한 규칙 없음 → `REVIEW_REQUIRED`

모집 종료(`CLOSED`)는 자격 판정과 별도로 표시한다.
`ELIGIBLE`도 공고 본문·첨부의 세부 조건(자유 텍스트) 확인을 전제로 한다.

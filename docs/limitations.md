# 알려진 한계

- **1차 판정은 구조화된 필드만 검사한다.** 지역(`supt_regin`), 업력(`biz_enyy`),
  신청 대상(`aply_trgt`) 등 API가 제공하는 범주 필드만 비교하며, 본문
  (`aply_trgt_ctnt` 등)에만 있는 세부 조건(추천 필수, 특정 프로그램 선정 기업
  한정 등)은 반영되지 않는다. `ELIGIBLE`도 반드시 본문을 확인해야 한다.
- **업력 산정 기준일이 API에 없다.** 접수 종료일 기준으로 명확히 미만인 경우만
  통과시키고, 경계 근처와 초과 추정은 자동 제외 없이 검토로 보낸다.
- **공식 자격 판정이 아니다.** 이 도구의 결과는 검토 우선순위 제안일 뿐이며,
  실제 지원 자격은 주관기관의 공고 원문과 문의를 통해 확인해야 한다.
- **API 데이터의 누락 또는 지연 가능성.** 공공데이터포털 API의 갱신 주기와
  커버리지는 보장되지 않으며, 모든 공고가 포함된다고 가정할 수 없다.
- **공고 본문과 첨부파일은 사람이 최종 확인해야 한다.** 자격 조건의 상당
  부분은 첨부파일(모집공고문)에만 존재할 수 있다.
- **회사 정보가 오래되거나 부정확할 수 있다.** 회사 데이터에는 기준일이
  있으며, 오래된 값으로 자동 탈락시키지 않는다.
- **모호한 정책 문구를 자동으로 확정할 수 없다.** 해석이 필요한 조건은
  `REVIEW_REQUIRED`로 보존하고 사람이 판단한다.
- **"지원 가능" 판정이 선정 가능성을 의미하지 않는다.** 자격 충족과 선정은
  별개의 문제다.
- **현재는 가상회사 데이터만 사용한다.** 실제 기업정보 수집과 판정은 별도
  승인 없이 수행하지 않는다.

## 발견 사항과 조치

### 1. `전남광주` 지역 매핑 누락 → 수정

표본에서 `supt_regin="전남광주"` 결합 표현 4건 관찰 (매핑표에 없어 REVIEW로
빠짐). 매핑표 groups에 `"전남광주": ["전남", "광주"]`를 추가해 명확한 판정
(서울 회사 기준 FAIL)으로 전환. 회귀 테스트 추가
(`tests/test_region_rule.py`).

### 2. 연령 규칙(지시서 14.4) 공백 → `age.v1` 구현

표본 10건이 청년 전용(`biz_trgt_age="만 20세 이상 ~ 만 39세 이하"`)인데
연령 규칙이 없어 다른 규칙만으로 판정되고 있었다. `age.v1`을 구현:

- 허용 구간이 성인 전 연령(만 20세 이상)을 포함하면 실질 제한 없음 → PASS
- 실질 제한이 있는데 대표자 생년월일이 없으면 → REVIEW
- 생년월일이 있으면 접수 기간 기준으로 계산하되, 연령 산정 기준일이
  데이터에 없으므로 자동 제외(FAIL)는 하지 않음

회귀 테스트 추가 (`tests/test_age_rule.py`,
`tests/test_evaluation.py::test_youth_only_announcement_is_review_not_eligible`).

### 3. 오제외(잘못된 INELIGIBLE) 점검 — 전수 확인

INELIGIBLE 34건 전건을 수동 대조한 결과, 필드 기준으로 잘못 제외된 사례는
발견되지 않았다 (지역 불일치 31건 중 대부분이 지역 한정 프로그램·입주 공고,
예비창업자 전용 3건, 대학생 전용 1건).

단, **창업보육센터 입주형 공고**(배재대·안양대·고려대기술지주 등)는 타지역
기업의 이전(입주)을 허용할 가능성이 있어 완전한 오제외 배제는 불가능하다.
지역 FAIL에는 항상 "이전 예정 기업 허용 문구 확인" 항목이 첨부되고 보고서에
이유와 함께 노출되므로, 사람이 최종 확인하는 것을 전제로 한다.

### 4. ELIGIBLE 과대판정 — 필드 판정의 설계상 한계 (알려진 오판 유형)

`ELIGIBLE` 55건 대조 결과, 본문에만 있는 제한 때문에 실제로는 지원이
어려운 사례들이 확인되었다. 이는 1차 필드 판정의 설계상 한계이며
`docs/limitations.md`에 명시되어 있다 (2차 상세 검토 단계의 근거 목록):

- 소속·자격 제한: 체육인 전용(178561), 센터 멤버십 회원 한정(178524),
  특정 시설 입주기업 한정(178569)
- 추천 필수: 수요기업 추천 기업만 신청 가능(178605)
- 업종·분야 제한: 핀테크(178534), 뷰티/농식품(178514), 콘텐츠(178592) —
  업종 판정은 지시서 15절에 따라 초기 MVP에서 의도적으로 제외
- 세부 연령 제한: "19세 이상 45세 이하"(178507) — `biz_trgt_age` 필드는
  3개 고정 구간뿐이라 45세 상한을 표현할 수 없음 (필드로는 감지 불가)

### 5. 신규 관찰 (api-observations.md 반영)

- `rcrt_prgs_yn="N"` 11건 관찰 — 마감 처리 정상 동작 확인
- `biz_trgt_age`는 3개 고정 토큰의 조합(표본에서 4가지 조합)
- `supt_regin` 결합 표현 존재 (`전남광주`)

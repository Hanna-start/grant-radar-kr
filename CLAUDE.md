# Grant Radar KR — 에이전트 작업 원칙

이 저장소에서 작업할 때 반드시 지켜야 하는 원칙이다.

## 절대 금지 (사용자 별도 승인 없이는 수행하지 않는다)

- 인증키(`KSTARTUP_API_KEY`) 출력, 커밋, 로그 노출. 표시가 필요하면 `ServiceKey=***`로 가린다.
- `.env` 커밋 (`.gitignore`로 제외되어 있는지 커밋 전 확인)
- 원격 GitHub 저장소 생성, 코드 푸시, 공개 업로드
- 실제 기업정보 수집·판정, 실제 이메일/Slack 발송, 유료 API 사용, 클라우드 배포
- 지원사업 자동 신청, CAPTCHA/로그인 우회, 무단 대량 수집, 첨부파일 재배포
- 근거 없는 자격 조건 생성, 자연어 모델 결과만으로 공고 자동 제외

## 핵심 판정 원칙

- 지원 가능한 공고를 놓치지 않는 것이 최우선. 정보 부족·모호함은 제외 사유가 아니라
  `REVIEW_REQUIRED` 사유다.
- 모든 판정 결과에는 근거(적용 규칙, 공고 값, 회사 값, 이유, 원문 URL)를 포함한다.
- API가 제공하지 않는 필드나 기능을 추측하지 않는다. 실제 응답을 확인한 후
  `docs/api-observations.md`에 기록하고 나서 모델을 결정한다.

## 개발 관행

- 테스트는 실제 API를 호출하지 않는다 (`httpx.MockTransport` 사용). 실제 인증키를
  테스트 코드에 넣지 않는다.
- 실행: `.venv\Scripts\python.exe -m grant_radar fetch`
- 테스트: `.venv\Scripts\python.exe -m pytest`
- 커밋은 작은 단위로 나눈다. 커밋 전 `.env` 미추적, 인증키 문자열 부재, 테스트 통과를 확인한다.
- 원본 API 응답(`data/raw/`)은 Git에서 제외한다.
